Что такое «система контроля версий»?

Система контроля версий (СКВ, или VCS — Version Control System) — это инструмент, который сохраняет историю изменений в файлах проекта, позволяя разработчикам отслеживать изменения и управлять версиями кода

К какому типу VSC относится? В чем главное отличие от других типов?

Version Control System (VSC): Тип: Это система, которая помогает отслеживать изменения в коде или документах. Главное отличие: VSC может быть централизованной (например, Subversion) или распределенной (например, Git). Главное отличие состоит в том, что распределённые системы позволяют каждому разработчику иметь полную копию репозитория, что упрощает работу и синхронизацию изменений между участниками.

В чем заключается преимущество использование Git?

Использование Git, системы управления версиями, имеет множество преимуществ: История изменений: Git сохраняет полную историю изменений вашего проекта, что позволяет отслеживать, кто, когда и какие изменения вносил. Это упрощает возврат к предыдущим версиям кода и понимание, когда и почему произошли определенные изменения. Ветвление и слияние: Git позволяет легко создавать и управлять ветками, что помогает разработчикам работать над новыми функциями или исправлениями ошибок в изолированном пространстве, не влияя на основную кодовую базу. Слияние веток позволяет объединять изменения и обновления.

Какие команды Git вы знаете?

git init — инициализация нового репозитория Git. git clone — копирование удаленного репозитория на локальный компьютер. git add — добавление изменений в индекс (staging area). git commit — сохранение изменений в локальном репозитории с комментарием. git status — отображение состояния рабочего каталога и индекса. git log — просмотр истории коммитов. git diff — отображение различий между файлами или коммитами. git branch — управление ветками (создание, удаление, перечисление веток).

В чем отличие Git и Git Hub?

Git: Git — это система контроля версий. Она позволяет разработчикам отслеживать изменения в коде, управлять различными версиями файлов и работать с несколькими участниками над одним проектом. Git можно использовать локально на своем компьютере, не требуя подключения к интернету. Он предоставляет команды для создания веток, слияния изменений, разрешения конфликтов и т.д. GitHub: GitHub — это веб-сервис, который предоставляет хостинг для репозиториев Git. Он позволяет пользователям хранить свои репозитории в облаке, делиться кодом с другими разработчиками и управлять проектами. GitHub предлагает дополнительные функции, такие как Pull Requests, Issues, Wikis, и другие инструменты для совместной работы. GitHub также поддерживает работу с репозиториями, созданными на Git, но предоставляет более удобный интерфейс для сотрудничества и управления проектами. В кратце: Git — это инструмент для управления версиями кода, а GitHub — это платформа для хостинга и совместной работы с кодом, использующая Git.
